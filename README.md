# Multi-Touch Attribution & Budget Optimization

## Overview
This project implements Last-Touch and Markov Chain attribution models within Databricks to evaluate the marketing channels (e.g., search ads, emails, referrals) in a customer journey. By quantifying the credit due to each channel, these models provide a data-driven framework for optimizing budget allocation and spend efficiency.

This project applies the recommended Medallion architecture, uses MLflow for artifacts tracking and Liquid Clustering for query optimization.

## Dataset
[Kaggle Multi-Touch Attribution](https://www.kaggle.com/datasets/vivekparasharr/multi-touch-attribution?resource=download)

## Methodology
Unlike standard implementations that treat a user's entire journey as one path, this analysis identifies journeys by conversion. Each conversion event counts as a distinct journey. Therefore, if a user converted more than once, each conversion is treated as the conclusion of a distinct, separate journey.

**Last-Touch Attribution**: This model assigns 100% of the credit to the final channel right before conversion. The attribution percentage is calculated as the simple ratio of a channel's total conversions against the total conversions across all channels.

**Markov Chain Attribution**: This model captures the value of assisting channels that drive mid-funnel engagement. These channels receive credit for a conversion even if they are not the final touchpoint before conversion. For the implementation, rather than using a recursive approach, a linear algebra approach (Matrix Inversion inspired by Absorbing Markov Chains) is used to calculate the removal effect, which quantifies how much each channel contributes to the total conversion probability.

### Main Takeaways
- Referral appears to be underfunded by the uniform budget. As the Markov Chain allocation is higher than both last-touch and uniform, we can say that Referral is secretly assisting user journeys and driving conversions more than what last-touch and uniform attribution credit it for.
- Both last-touch and Markov Chain propose lower budgets for Search Ads. This indicates that Search Ads are currently less effective at both leading straight to conversions and assisting the earlier stages of a user journey.
- Last-touch allocates more budget to Email than Markov Chain does. This implies that emails are less of a tool for creating new demand or awareness, but rather a better tool for leading straight to the final conversion.

### Final Decision Strategy
If the priority is on immediate conversion, prioritize following the last-touch budget allocation. Otherwise, it is recommended to follow the Markov Chain allocation.
